# 16. Видеоплатформа — Детальная спецификация

Вертикальная видеолента с shoppable видео, интеграцией маркетплейса и мессенджера.

> **Реализация:** Phase 7. Архитектурный фундамент закладывается в Phase 0.

---

## 1. Концепция

### Философия

Ninhao Video — это **commerce-driven контент**, а не entertainment-driven. Каждое видео потенциально ведёт к покупке. Создатели зарабатывают не от просмотров, а от конверсий. Это обеспечивает качественный, полезный контент вместо кликбейта.

### Принципы

1. **Shoppable-first** — любой товар в видео можно купить в один тап
2. **Quality over quantity** — алгоритм приоритезирует конверсии, а не время просмотра
3. **Creator = Seller** — продавцы маркетплейса это и есть создатели контента
4. **Social discovery** — рекомендации на основе того, что смотрят и покупают друзья
5. **Respectful UX** — опциональный digital wellbeing: таймер, daily digest

## 4. Видеоплеер

### 4.1 Технология

- **expo-av** или **react-native-video** для воспроизведения
- **HLS** (HTTP Live Streaming) для adaptive bitrate
- Preload: предзагрузка 2 следующих видео для мгновенного переключения

### 4.2 Качество (adaptive bitrate)

| Качество | Разрешение | Bitrate | Когда |
|----------|-----------|---------|-------|
| Low | 480p | 1 Mbps | Медленный интернет / data saver |
| Medium | 720p | 2.5 Mbps | WiFi / 4G |
| High | 1080p | 5 Mbps | WiFi |
| Original | Up to 4K | 10+ Mbps | Только WiFi, opt-in |

### 4.3 Поведение плеера

| Ситуация | Действие |
|---------|---------|
| Видео в viewport | Автоплей (muted при первом запуске) |
| Свайп на другое видео | Пауза текущего, плей нового |
| Фоновый режим (home button) | Пауза |
| Звонок | Пауза, resume после звонка |
| Наушники отключены | Пауза |
| Нет сети | Показать cached frame + "Нет подключения" |
| Конец видео | Loop (повтор) |

## 7. Алгоритм ленты "Для вас"

### 7.1 Сигналы

| Сигнал | Вес | Описание |
|--------|-----|----------|
| **Conversion rate** | Высокий | Видео с высоким % добавлений в корзину |
| **Watch time %** | Высокий | Досмотренность (>75% = хорошо) |
| **Friends watched** | Высокий | Друзья из Кругов смотрели/лайкали |
| **Engagement** | Средний | Лайки, комментарии, шеры |
| **Freshness** | Средний | Новые видео получают boost |
| **Creator quality** | Средний | Рейтинг создателя, верификация |
| **User interests** | Средний | На основе истории покупок и просмотров |
| **Diversity** | Низкий | Разнообразие категорий (не только одна тема) |

### 7.2 Anti-patterns (что НЕ продвигаем)

- Clickbait thumbnails без содержания
- Видео без товаров (контент ради контента — не в приоритете)
- Одинаковый контент от разных создателей (spam detection)
- Контент с нарушениями (nudity, violence, fraud)

### 7.3 "Подписки" — лента

Хронологическая лента видео от создателей, на которых подписан пользователь. Без алгоритмической обработки.

## 10. Video Service (Go)

### 10.1 Pipeline обработки

```
Upload → Validation → Transcoding → Thumbnail → HLS → CDN → Ready
  │          │             │            │         │        │
  └── S3     └── Check     └── FFmpeg   └── Extract└── Segment└── Notify
     (raw)      format,       (480p,       frame      .m3u8     (WebSocket
                size,         720p,        at 1s      + .ts      event)
                duration      1080p)
                ≤60s,
                ≤500MB)
```

### 10.2 Хранение

```
S3/MinIO:
  /videos/
    /{video_id}/
      raw.mp4              # Оригинал (временно, удаляется после транскодинга)
      thumb.jpg            # Thumbnail
      poster.jpg           # Poster (первый кадр)
      480p/
        index.m3u8
        segment_000.ts
        segment_001.ts
      720p/
        index.m3u8
        ...
      1080p/
        index.m3u8
        ...
      master.m3u8          # Master playlist (adaptive)
```

### 10.3 Модели данных (бекенд)

```sql
-- Видео
videos (
  id UUID PRIMARY KEY,
  creator_id UUID REFERENCES users,
  title VARCHAR(200),
  description TEXT,
  duration INT,                    -- секунды
  status: processing | ready | failed | removed,
  video_url TEXT,                  -- HLS master playlist URL
  thumbnail_url TEXT,
  poster_url TEXT,
  like_count INT DEFAULT 0,
  comment_count INT DEFAULT 0,
  share_count INT DEFAULT 0,
  view_count INT DEFAULT 0,
  visibility: public | followers,
  allow_comments BOOLEAN DEFAULT true,
  allow_duets BOOLEAN DEFAULT true,
  hashtags TEXT[],
  created_at TIMESTAMP,
  published_at TIMESTAMP
)

-- Привязка товаров к видео
video_product_tags (
  id UUID PRIMARY KEY,
  video_id UUID,
  product_id UUID,
  position_x FLOAT,               -- 0.0-1.0 (% от ширины)
  position_y FLOAT,               -- 0.0-1.0 (% от высоты)
  display_from INT,               -- секунда начала показа
  display_until INT,              -- секунда конца показа (null = всё видео)
  sort_order INT
)

-- Лайки
video_likes (
  video_id UUID,
  user_id UUID,
  created_at TIMESTAMP,
  PRIMARY KEY (video_id, user_id)
)

-- Комментарии
video_comments (
  id UUID PRIMARY KEY,
  video_id UUID,
  user_id UUID,
  parent_id UUID,                 -- reply (1 уровень)
  content TEXT,
  like_count INT DEFAULT 0,
  created_at TIMESTAMP
)

-- Подписки
creator_follows (
  creator_id UUID,
  follower_id UUID,
  created_at TIMESTAMP,
  PRIMARY KEY (creator_id, follower_id)
)
```

## 11. Content Moderation

### 11.1 Автоматическая (AI)

| Проверка | Когда | Действие |
|---------|-------|---------|
| NSFW detection | При загрузке | Блокировка + уведомление |
| Violence detection | При загрузке | Блокировка |
| Copyright music | При загрузке | Предупреждение |
| Spam detection | При публикации | Понижение в ленте |
| Text detection (OCR) | При загрузке | Проверка на запрещённые тексты |

### 11.2 Ручная

- Кнопка "Пожаловаться" на каждом видео
- Очередь модерации в Admin panel (Filament)
- Категории: spam, nudity, violence, scam, copyright, other

## 12. Digital Wellbeing

### 12.1 Фичи (opt-in)

| Фича | Описание |
|------|---------|
| **Daily timer** | Напоминание после N минут просмотра (настраиваемо) |
| **Break reminder** | "Вы смотрите 20 минут. Перерыв?" |
| **Daily digest** | Вместо бесконечной ленты: 10 лучших видео за день |
| **Bedtime mode** | Затемнение + чёрно-белый после указанного времени |
| **Usage stats** | Сколько минут/день, сколько видео, сколько покупок из видео |

### 12.2 Зачем

- Дифференциация от TikTok (который критикуют за addiction)
- Повышение доверия пользователей
- Более качественное время = лучшие конверсии (меньше zombie scrolling)
