# 08. Корзина, оформление и заказы

Экраны корзины, оформления заказа, списка заказов и деталей заказа.

---

## 1. Корзина (CartScreen)

**Маршрут:** `/(tabs)/cart`
**Хранилище:** Zustand (локальное, без серверного API)
**Auth:** Нет

### Layout

```
┌─────────────────────────────────┐
│  Корзина (3)                   │
├─────────────────────────────────┤
│  ☑ Выбрать все                 │
├─────────────────────────────────┤
│  ☑ ┌────┐ Product Name         │
│    │Фото│ $99.99               │
│    └────┘ [-] 2 [+]            │  ← Quantity controls
│                        🗑      │  ← Swipe left или кнопка
│  ─────────────────────────     │
│  ☑ ┌────┐ Another Product      │
│    │Фото│ $49.99               │
│    └────┘ [-] 1 [+]            │
│  ─────────────────────────     │
│  ☐ ┌────┐ Deselected Item      │  ← Не выбран для покупки
│    │Фото│ $29.99               │
│    └────┘ [-] 1 [+]            │
│                                 │
├─────────────────────────────────┤
│  Итого (2 товара): $249.97     │
│  [    Оформить заказ         ] │  ← primary button
└─────────────────────────────────┘
```

### Поведение

- **Checkbox per item** — выбор товаров для текущей покупки (как AliExpress)
- **"Выбрать все"** — toggle all checkboxes
- **Quantity** — кнопки +-. Min: 1, Max: stock_quantity. Haptic при изменении
- **Удаление** — swipe left на элементе → красная кнопка "Удалить"
- **Итого** — сумма только выбранных товаров
- **Оформить** — только если выбран хотя бы 1 товар. Если не авторизован → redirect на login
- **Empty state** — "Корзина пуста" + кнопка "Перейти к покупкам"
- **Persistence** — Zustand persist в MMKV. Корзина сохраняется между сессиями
- **Аналитика:** `CheckoutStarted` при нажатии "Оформить"

### Cart Item Component

```
┌─────────────────────────────────────┐
│ ☑  ┌──────┐  Product Title Long    │
│    │      │  Name Here             │
│    │ Фото │  $99.99                │
│    │      │  ★ 4.5                 │
│    └──────┘  [-]  2  [+]          │
└─────────────────────────────────────┘
```

- Фото: 80x80, radius-md
- Тап на фото/название → Product Detail
- Тап на checkbox → toggle selection

## 2. Оформление заказа (CheckoutScreen)

**Маршрут:** `/checkout`
**API:** `POST /orders`, `GET /addresses`
**Auth:** JWT (обязательно)

### Layout

```
┌─────────────────────────────────┐
│  ← Оформление заказа           │
├─────────────────────────────────┤
│                                 │
│  📍 Адрес доставки              │
│  ┌──────────────────────────┐  │
│  │ John Doe                  │  │
│  │ 123 Main St, New York    │  │
│  │ +1234567890          ✏  │  │  ← Тап → выбор адреса
│  └──────────────────────────┘  │
│  или [+ Добавить адрес]       │
│                                 │
│  📦 Товары                      │
│  ┌────┐ Product Name × 2      │
│  └────┘ $199.98               │
│  ┌────┐ Another × 1           │
│  └────┘ $49.99                │
│                                 │
│  💬 Комментарий к заказу       │
│  ┌──────────────────────────┐  │
│  │ Пожалуйста, упакуйте... │  │
│  └──────────────────────────┘  │
│                                 │
│  ─────────────────────────     │
│  Товары (3):         $249.97   │
│  Доставка:           Бесплатно │
│  ─────────────────────────     │
│  Итого:              $249.97   │
│                                 │
├─────────────────────────────────┤
│  [     Оплатить $249.97      ] │  ← primary button
└─────────────────────────────────┘
```

### Поведение

- **Адрес** — показать default адрес из `GET /addresses`. Если нет адресов → "Добавить адрес"
- **Выбор адреса** — модальный список адресов с radio buttons
- **Товары** — readonly список из корзины (только выбранные)
- **Комментарий** — опциональное текстовое поле
- **Кнопка "Оплатить"** — `POST /orders` с данными. При успехе:
  1. Удалить купленные товары из корзины
  2. Перейти к деталям заказа
  3. Показать toast "Заказ создан"
  4. Отправить `PaymentClicked`
- **Ошибки валидации** — показать под соответствующими полями
- **Loading state** — кнопка disabled + spinner при отправке

### Выбор адреса (Bottom Sheet)

```
┌─────────────────────────────────┐
│  Выберите адрес          + New  │
├─────────────────────────────────┤
│  ◉ Home                        │
│    John Doe, 123 Main St       │
│    New York, US 10001          │
│  ─────────────────────         │
│  ○ Office                      │
│    John Doe, 456 Work Ave      │
│    New York, US 10002          │
└─────────────────────────────────┘
```

## 3. Список заказов (OrdersScreen)

**Маршрут:** `/(tabs)/orders`
**API:** `GET /orders`
**Auth:** JWT

### Layout

```
┌─────────────────────────────────┐
│  Мои заказы                    │
├─────────────────────────────────┤
│  ┌──────────────────────────┐  │
│  │ Заказ #uuid-short        │  │
│  │ 12 марта 2026            │  │
│  │ ┌────┐ ┌────┐            │  │  ← Превью фото товаров (до 3)
│  │ └────┘ └────┘            │  │
│  │ 2 товара    $249.97      │  │
│  │ [Оплачен] ●              │  │  ← Status badge
│  └──────────────────────────┘  │
│                                 │
│  ┌──────────────────────────┐  │
│  │ Заказ #uuid-short        │  │
│  │ 10 марта 2026            │  │
│  │ 1 товар     $99.99       │  │
│  │ [Доставлен] ●            │  │
│  └──────────────────────────┘  │
│                                 │
│  ...                           │
└─────────────────────────────────┘
```

### Поведение

- Пагинация (infinite scroll или load more)
- Pull-to-refresh
- Тап на заказ → Order Detail
- Status badge с цветом:
  - pending → yellow
  - paid → blue
  - processing → blue
  - shipped → blue
  - delivered → green
  - cancelled → gray
  - refunded → red
- Auth guard: если не авторизован → AuthPrompt
- Empty state: "У вас пока нет заказов"

## 4. Детали заказа (OrderDetailScreen)

**Маршрут:** `/order/[id]`
**API:** `GET /orders/{id}`
**Auth:** JWT

### Layout

```
┌─────────────────────────────────┐
│  ← Заказ #uuid-short           │
├─────────────────────────────────┤
│                                 │
│  Статус: [Оплачен] ●          │
│  Создан: 12 марта 2026        │
│                                 │
│  ─────────────────────────     │
│  Товары                        │
│  ┌────┐ Product Name × 2      │
│  │Фото│ $99.99 × 2 = $199.98  │  ← Тап → ProductDetail
│  └────┘ Seller: Company A      │
│  ┌────┐ Another × 1           │
│  │Фото│ $49.99                 │
│  └────┘ Seller: Company B      │
│                                 │
│  ─────────────────────────     │
│  Адрес доставки                │
│  John Doe                      │
│  123 Main St, New York 10001   │
│  +1234567890                   │
│                                 │
│  ─────────────────────────     │
│  Оплата                        │
│  Метод: Demo                   │
│  Статус: Оплачен              │
│  Transaction: TXN-uuid         │
│                                 │
│  ─────────────────────────     │
│  Итого                         │
│  Товары (3):       $249.97     │
│  Доставка:         Бесплатно   │
│  Итого:            $249.97     │
│                                 │
│  ─────────────────────────     │
│  Действия                      │
│  [Оплатить]     ← если pending │
│  [Подтвердить доставку]        │  ← если shipped
│  [Связаться с поддержкой]      │  ← создать тикет с order_id
│                                 │
└─────────────────────────────────┘
```

### Действия по статусу

| Статус заказа | Доступные действия |
|--------------|-------------------|
| pending | "Оплатить" (`POST /orders/{id}/pay`) |
| paid | — |
| processing | — |
| shipped | "Подтвердить доставку" (`POST /orders/{id}/confirm-delivery`) |
| delivered | "Оставить отзыв" (→ ProductDetail) |
| cancelled | — |
| refunded | — |

## 5. Mock Payment Flow

В MVP оплата — демо-режим:

```
1. Нажать "Оплатить" на экране заказа
2. Показать confirmation bottom sheet:
   "Оплатить заказ #xxx за $249.97?"
   [Оплатить] [Отмена]
3. POST /orders/{id}/pay
4. При успехе: обновить статус → "paid", показать toast
5. При ошибке: показать error message
```

## 6. Связь с другими экранами

| Из | В | Действие |
|----|---|---------|
| Cart → Checkout | Login | Если не авторизован |
| Checkout → Addresses | Address List | Выбор адреса |
| Order Detail → Product | Product Detail | Тап на товар |
| Order Detail → Support | Create Ticket | "Связаться с поддержкой" (order_id) |
| Orders tab → Auth | Login | Если не авторизован |
